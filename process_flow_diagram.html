Phase 2: Query Search (Multimodality)

Every mode converts user input into a 1024-dim query vector that matches the indexed product format.
Gemma structures the query into: Description · Color · Pattern · Material · Length · Sleeve · Silhouette · Niche · Occasion · Season · Style tags

Mode A: Image Only

Input: Upload fashion design image

Step 1 → Rekognition extracts labels
(garment type, pattern, fabric — used for insights only, NOT in embedding)

Step 2 → Gemma Vision analyses the image
Extracts structured attributes in indexed format:
Description: Black floral print long sleeve dress
Color: Black · Pattern: Blue floral print
Material: Silk · Length: Midi · Sleeve: Long sleeve
Silhouette: A-line · Niche: Casual Wear
Occasion: Daily · Season: Summer · Style tags: … Step 3 → Titan Hybrid Embedding
inputImage = uploaded photo
inputText = Gemma structured output
→ 1024-dim query vector

Mode B: Image + Text

Input: Upload image + describe intent
e.g. "Orange summer skirt with floral top"

Step 1 → Rekognition extracts labels
(insights only, NOT in embedding)

Step 2 → Gemma Vision analyses image + user description together
Fuses visual analysis with user intent — user text steers ambiguous fields:
Description: Orange floral sleeveless co-ord set
Color: Orange ← user said "orange"
Pattern: Floral print · Material: Cotton
Length: Midi · Sleeve: Sleeveless
Silhouette: A-line · Niche: Casual Wear · … Step 3 → Titan Hybrid Embedding
inputImage = uploaded photo
inputText = Gemma fused output (user text is NOT added separately)
→ 1024-dim query vector

Mode C: Text Only

Input: Type description only (no image)
e.g. "red party dress"

Step 1 → Gemma text-only imagines a full product listing
Also classifies query as Generic or Detailed:
Query type: Generic
Color: Red · Pattern: None · Material: Satin
Length: Mini · Sleeve: Sleeveless
Silhouette: Bodycon · Niche: Party Wear · … Step 2 → Titan Text-Only Embedding
inputText = raw user text + Gemma structured output combined
(raw text "red party dress" preserved as keyword safety net)
→ 1024-dim query vector

Step 3 → Search strategy depends on query type:
Generic → kNN + BM25 keyword boost on niche, occasion, style tags
Detailed → kNN only (BM25 skipped to avoid noise)
